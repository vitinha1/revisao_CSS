A Regra da Cascata

Quando existem múltiplas regras de estilo para o mesmo elemento, o navegador precisa decidir qual delas aplicar. Essa decisão segue a Regra da Cascata, que considera três critérios principais: 1. Origem e importância Estilos do navegador (default) → menos importantes Estilos do desenvolvedor (no CSS) → prioridade normal Estilos com !important → prioridade máxima.

Se duas regras tiverem a mesma importância e especificidade, vale a última lida pelo navegador (de baixo para cima no CSS).

A Regra da Cascata no CSS segue esta ordem de prioridade: !important Especificidade do seletor (inline > id > classe > tag) Ordem no código (última regra ganha)

Especificidade no CSS

A especificidade no CSS é o sistema de “pontuação” que o navegador usa para decidir qual regra aplicar quando várias regras competem pelo mesmo elemento.

Como funciona a especificidade?

Cada tipo de seletor recebe um peso. Quanto mais específico o seletor, maior a chance dele “ganhar” na disputa. Pontuação básica: Inline (style="...") → 1000 pontos ID (#menu) → 100 pontos Classe, atributo, pseudo-classe (.btn, [type="text"], :hover) → 10 pontos Elemento e pseudo-elemento (div, p, ::before) → 1 ponto Universal (*) → 0 pontos
