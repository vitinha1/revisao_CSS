A Regra da Cascata

Quando existem múltiplas regras de estilo para o mesmo elemento, o navegador precisa decidir qual delas aplicar. Essa decisão segue a Regra da Cascata, que considera três critérios principais: 1. Origem e importância Estilos do navegador (default) → menos importantes Estilos do desenvolvedor (no CSS) → prioridade normal Estilos com !important → prioridade máxima.

Se duas regras tiverem a mesma importância e especificidade, vale a última lida pelo navegador (de baixo para cima no CSS).

A Regra da Cascata no CSS segue esta ordem de prioridade: !important Especificidade do seletor (inline > id > classe > tag) Ordem no código (última regra ganha)

Especificidade no CSS

A especificidade no CSS é o sistema de “pontuação” que o navegador usa para decidir qual regra aplicar quando várias regras competem pelo mesmo elemento.

Como funciona a especificidade?

Cada tipo de seletor recebe um peso. Quanto mais específico o seletor, maior a chance dele “ganhar” na disputa. Pontuação básica: Inline (style="...") → 1000 pontos ID (#menu) → 100 pontos Classe, atributo, pseudo-classe (.btn, [type="text"], :hover) → 10 pontos Elemento e pseudo-elemento (div, p, ::before) → 1 ponto Universal (*) → 0 pontos

Combinação de Seletores

combinação de seletores no CSS permite que você escreva regras mais precisas e específicas, aplicando estilos apenas quando certas condições de relação entre elementos forem atendidas. Existem 5 tipos principais de combinadores no CSS:

1. Descendente (espaço)

Seleciona elementos dentro de outro.

div p { color: blue; }

Afeta todos os p> que estão dentro de um div>, em qualquer nível.

2. Filho direto (>)

Seleciona apenas o filho imediato.

div > p { color: red; }

Afeta só os

que são filhos diretos do div>, e não os netos.

3. Irmão adjacente (+)

Seleciona o elemento logo após outro.

h1 + p { color: green; }

Afeta o p> que vem imediatamente depois de um h1>

4. Irmãos gerais (~)

Seleciona todos os irmãos seguintes.

h1 ~ p { color: orange; }

Afeta todos os p> que estão depois de um h1>, no mesmo nível.

5. Agrupamento de seletores (,)

Aplica o mesmo estilo a múltiplos seletores.

h1, h2, h3 { color: purple; }

Afeta todos os títulos h1, h2 e h3. Exemplo prático

Pseudo-classes Estruturais

Principais Pseudo-classes Estruturais

:first-child Seleciona o primeiro filho de um elemento pai. Exemplo: o primeiro parágrafo dentro de uma seção.

:last-child Seleciona o último filho de um elemento pai. Exemplo: o último item de uma lista.

:nth-child(n) Seleciona o n-ésimo filho de um elemento pai. Exemplo: todo segundo ou terceiro item de uma lista.

:nth-last-child(n) Seleciona o n-ésimo filho contando de trás para frente. Exemplo: penúltimo item de uma lista.

:only-child Seleciona o único filho de um elemento pai. Exemplo: um parágrafo que é o único dentro de uma div.

:first-of-type Seleciona o primeiro elemento de um tipo específico entre os irmãos. Exemplo: o primeiro parágrafo dentro de uma seção, mesmo que haja outros tipos de elementos antes.

:last-of-type Seleciona o último elemento de um tipo específico entre os irmãos.

:nth-of-type(n) Seleciona o n-ésimo elemento de um tipo específico. Exemplo: o terceiro parágrafo, ignorando outros tipos de elementos.

:nth-last-of-type(n) Seleciona o n-ésimo elemento de um tipo específico contando de trás para frente.

:only-of-type Seleciona o único elemento de um tipo específico entre os irmãos.

Compreendendo o Display

A propriedade display define como um elemento é exibido na página e como ele interage com outros elementos ao redor. Ela determina se o elemento: Ocupa uma linha inteira ou apenas o espaço necessário. Pode ter largura e altura definidas. Fica lado a lado de outros elementos ou em linha.

>Principais valores de display

block (bloco) O elemento ocupa toda a largura disponível. Sempre começa em uma nova linha. Ex.: div>, h1>, p> (por padrão). inline (linha) O elemento ocupa apenas o espaço necessário para seu conteúdo. Não quebra linha; fica lado a lado de outros elementos. Ex.: span>, a>, strong> (por padrão). inline-block Combina características de inline e block: Fica lado a lado de outros elementos (como inline). Permite definir largura e altura (como block). none O elemento não é exibido na página. Não ocupa espaço algum. flex Define um container flexível que organiza seus filhos de forma dinâmica. Facilita alinhamento e distribuição de espaço. grid Define um container em grade para posicionar os elementos em linhas e colunas.

Elementos Inline e Block

Elementos Block (bloco) Ocupam toda a largura disponível da linha. Sempre começam em uma nova linha. Podemos definir largura e altura. Ex.: div>, p>, h1>, section>

Características principais:

Criam “blocos” visíveis na página. Usados para organizar seções e conteúdos maiores.

Elementos Inline (linha)

Ocupam apenas o espaço necessário para o conteúdo. Não quebram a linha; ficam lado a lado de outros elementos. Não podemos definir largura e altura diretamente. Ex.: span>, a>, strong>, em>

Características principais:

Servem para formatar trechos de texto ou pequenos elementos dentro de blocos. Usados para detalhes ou partes internas do conteúdo.

Espaçamento com Box Model

Margin

spaço externo: não afeta o conteúdo nem o tamanho interno da caixa. Pode ser individual ou geral: você pode definir margens para cima, baixo, esquerda e direita separadamente. Valores possíveis: pode ser em pixels (px), porcentagem (%), em, ou auto (útil para centralizar elementos horizontalmente).

Padding

Espaço interno: afasta o conteúdo da borda da caixa, sem afetar os elementos externos. Pode ser individual ou geral: você pode definir padding para cima, baixo, esquerda e direita separadamente. Valores possíveis: pixels (px), porcentagem (%), em ou rem.

Border

Fica entre o padding e a margin: envolve o conteúdo e o espaço interno (padding) da caixa, mas está dentro da margem. Pode ter estilo, cor e espessura: você pode definir se é sólida, tracejada, pontilhada, dupla, etc. Pode ser aplicada de forma geral ou em lados específicos: topo, direita, inferior ou esquerda.
